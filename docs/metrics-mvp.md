# Блок 8 — Метрики MVP и критерии успеха

> Версия 1.0 — 26.06.2026

---

## Главный вопрос блока

**Что значит «гипотеза подтверждена»?**

Гипотеза: аудитория (Техник + Сломавшийся) платит за адаптивный звуковой протокол — не за плейлист, а за систему с обратной связью — и возвращается достаточно часто, чтобы бизнес был жизнеспособен.

Ответ — в трёх числах: конверсия, удержание, CAC.

---

## Воронка целиком

```
Просмотр ролика (VK Клипы / Instagram / TikTok)
    ↓  CTR в биографию/ссылку: 1–3%
Лендинг
    ↓  Регистрация на триал: 8–15%
Триал (3 ночи, полный доступ)
    ↓  Конверсия в оплату: 20–30%
Платный подписчик
    ↓  Удержание месяц 1: >60%
    ↓  Удержание месяц 3: >35%
```

## Метрики по уровням

### Уровень 1 — Контент и трафик

| Метрика | Целевое значение | Что значит если ниже |
|---------|-----------------|---------------------|
| Просмотры ролика до конца (completion rate) | >40% | Крючок слабый — переделать начало |
| CTR ссылка в био / sticker | 1–3% | CTA в ролике неубедительный |
| Стоимость клика на лендинг | <15 ₽ | Платный трафик нерентабелен без роста |

### Уровень 2 — Лендинг и триал

| Метрика | Целевое значение | Что значит если ниже |
|---------|-----------------|---------------------|
| Landing → регистрация (trial start rate) | 8–15% | Лендинг не закрывает барьеры; тестировать хедлайн и CTA |
| Стоимость триала (CPT) | <100 ₽ | Платный трафик нерентабелен |
| Кол-во сессий за 3 дня триала (среднее) | ≥2 | Если 0–1 — онбординг не работает, первая ночь не случилась |
| Открытие хотя бы одной сессии | >70% триалов | Если ниже — проблема в первом шаге, не в продукте |

### Уровень 3 — Конверсия в оплату

| Метрика | Целевое значение | Сигнал тревоги |
|---------|-----------------|----------------|
| Trial → Paid (конверсия) | **20–30%** | <10% после 100 триалов → менять оффер или триал |
| Выбор годового тарифа | >30% платящих | Если мало — поднять акцент на скидке 40% |
| Стоимость платящего подписчика (CAC) | **<200 ₽** | >500 ₽ → unit economics не сходится на старте |

> **CAC считать как:** (расходы на контент + продвижение) / кол-во новых платящих за период.  
> Своё время на контент включать по ставке — иначе CAC будет казаться нулевым.

### Уровень 4 — Удержание (Retention)

| Метрика | Целевое значение | Сигнал тревоги |
|---------|-----------------|----------------|
| Сессий в неделю (активный пользователь) | ≥3 | <1 в неделю → нет привычки, удержание провалится |
| D30 retention (месячная когорта) | **>40%** | <25% → продукт не формирует привычку |
| D90 retention | >25% | Ключевой показатель для годовой подписки |
| Ежемесячный churn | **<10%** | >15% → бизнес не сходится даже при хорошем CAC |
| Продление (Месяц → Месяц 2) | >60% | Если <50% — бот-напоминания не работают |

### Уровень 5 — Экономика

| Метрика | Значение |
|---------|---------|
| ARPU месячный | 490 ₽ × (доля месячных) + 290 ₽ × (доля годовых, в пересчёте) |
| LTV при churn 8%/мес | ~490 / 0.08 = **≈6 100 ₽** |
| LTV при churn 5%/мес | ~490 / 0.05 = **≈9 800 ₽** |
| LTV/CAC (целевое) | **>10** при CAC 200 ₽ и churn 8% |
| Payback period (окупаемость CAC) | **1 месяц** при CAC <490 ₽ |
| Точка безубыточности (инфраструктура) | **2–3 подписчика** (VPS ~925 ₽/мес ÷ 476 ₽ чистых с подписки) |
| Порог достаточных данных | **50 подписчиков** — когорта, на которой Trial→Paid и D30 статистически значимы |

## Критерии «гипотеза подтверждена»

Все три условия одновременно:

```
✓  Trial → Paid > 20%   (продукт закрывает потребность)
✓  D30 retention > 35%  (привычка формируется)
✓  CAC < 200 ₽          (трафик масштабируется)
```

Если все три выполнены на когорте **50+ платящих подписчиков** — гипотеза подтверждена, можно вкладывать в рост.

> Инфраструктура окупается уже с 2–3 подписчиков (VPS ~925 ₽/мес, чистый доход с подписки ~476 ₽). 50 — это порог достаточных данных, не финансового выживания.

---

## Критерии «стоп — нужен пивот»

Любое из условий после **100 завершённых триалов:**

```
✗  Trial → Paid < 10%   → проблема в продукте или оффере
✗  D30 retention < 20%  → привычка не формируется
✗  CAC > 600 ₽          → трафик не масштабируется
✗  <2 сессий за триал (медиана) → онбординг сломан
```

При стоп-сигнале — диагностика: слушаем запись первой сессии протокола, читаем ответы анкеты «до/после», смотрим на каком шаге уходят.

## Минимальная аналитика на MVP

Без платных аналитических сервисов. Всё из своей БД.

### SQL-запросы которые нужны с первого дня

```sql
-- Конверсия trial → paid (за последние 30 дней)
SELECT
  COUNT(*) FILTER (WHERE subscription_status IN ('active', 'expired') AND trial_started_at > now() - interval '30 days') AS trials,
  COUNT(*) FILTER (WHERE subscription_status = 'active' AND trial_started_at > now() - interval '30 days') AS paid,
  ROUND(100.0 * COUNT(*) FILTER (WHERE subscription_status = 'active') /
    NULLIF(COUNT(*) FILTER (WHERE trial_started_at > now() - interval '30 days'), 0), 1) AS conversion_pct
FROM users;

-- Медиана сессий за триал
SELECT percentile_cont(0.5) WITHIN GROUP (ORDER BY session_count) AS median_sessions
FROM (
  SELECT u.id, COUNT(sl.id) AS session_count
  FROM users u
  LEFT JOIN sessions_log sl ON sl.user_id = u.id
    AND sl.played_at BETWEEN u.trial_started_at AND u.trial_started_at + interval '3 days'
  WHERE u.trial_started_at > now() - interval '60 days'
  GROUP BY u.id
) t;

-- D30 retention (когорта за прошлый месяц)
SELECT
  COUNT(*) FILTER (WHERE subscription_status = 'active') AS still_active,
  COUNT(*) AS total_paid,
  ROUND(100.0 * COUNT(*) FILTER (WHERE subscription_status = 'active') / COUNT(*), 1) AS d30_retention
FROM users
WHERE subscription_status IN ('active', 'expired')
  AND trial_started_at BETWEEN now() - interval '60 days' AND now() - interval '30 days';
```

### Что смотреть каждую неделю

```
Пн:  Новые триалы за неделю
     Конверсия trial→paid (текущая)
     Сессий у активных пользователей за неделю

Пт:  Churn (кто не продлил из тех, у кого истёк)
     Топ-3 трека по количеству запусков
     Ответы анкеты «что мешает» (из протокола)
```

### Что НЕ смотреть на MVP

- DAU/MAU в абсолютных числах (при <200 пользователях — бессмысленно)
- Bounce rate лендинга (нет достоверных данных без Яндекс.Метрики)
- NPS (слишком рано, нет статистики)

---

## Когда добавлять Яндекс.Метрику

После 500+ уникальных посетителей лендинга в месяц. До этого — шум перевешивает сигнал. На MVP достаточно UTM-меток в ссылках на лендинг для разбивки трафика по платформам.

```
https://dreif.ru/?utm_source=vk&utm_medium=clips&utm_campaign=sleep_v1
https://dreif.ru/?utm_source=instagram&utm_medium=reels&utm_campaign=focus_v2
```

Сохраняем `utm_source` при регистрации → в `users.source` → считаем конверсию по источнику напрямую из БД.

## Ориентиры рынка для сравнения

| Продукт | Trial→Paid | Churn/мес | LTV/CAC |
|---------|-----------|-----------|---------|
| Brain.fm (аналог) | ~25% | ~7% | >15 |
| Calm | ~12% (широкая аудитория) | ~5% | >20 |
| Типичный B2C wellness SaaS | 10–20% | 5–12% | 5–15 |
| **Дрейф — цель** | **20–30%** | **<10%** | **>10** |

Brain.fm — ближайший аналог по механике. Достижимо при правильном онбординге.

## Порядок действий после запуска

| Период | Что смотрим | Что меняем |
|--------|-------------|------------|
| Дни 1–7 | Открывают ли первую сессию в триале? | Онбординг (первое письмо бота) |
| Дни 8–21 | Конверсия в оплату | Конверсионное сообщение день 3 |
| Месяц 1–2 | D30 retention | Частота сессий, напоминания |
| Месяц 3+ | Churn, CAC по источникам | Масштабирование лучшего канала |
