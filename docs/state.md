## Что уже зафиксировано

### Блок 1 — Бренд и позиционирование ✓
- Название: Audium (проверено — в нише wellness/аудио в РФ свободно; audi-um.ru в другой нише)
- Цвета: фон #0B1623, карточки #142233, акцент #7DD8CC, CTA #D4A857, текст #E7E3D8 / #9CA3AE
- Тёмная тема — основной режим, не опциональный
- Бренд-персонаж: отсутствует (полностью безликий продукт)

### Блок 2 — Продуктовая линейка ✓
- Один платный тариф: 490 ₽/мес, 3 490 ₽/год
- Пробный период: 3 дня («три ночи»), полный доступ, без ввода карты
- Постоянного бесплатного тарифа нет; на лендинге — превью-клипы 60–90 сек
- Каталог по кейсам: Сон / Фокус / Снятие стресса / Восстановление
- 15–20 треков на старте; контент пополняется по кейсам, не по расписанию
- Удержание — через протокол и прогрессию, не через новый контент

### Исследование аудитории ✓
Три сегмента:
- **Техник** (25–35, IT/инженерия): биохакер, читает pubmed, платит за механизм и эффективность
- **Сломавшийся** (28–45, стресс/выгорание): хочет «выключить голову», покупает эмоционально
- **Ищу ответ** (22–35, нейро-энтузиаст): слышал о биноуральных битах, не уверен, нужно объяснение + соц-доказательство

Пять барьеров покупки (и как закрываем):
1. «Не работает» → технический блок с механизмом и частотами
2. «Небезопасно» → честный FAQ с противопоказаниями (не замалчивать)
3. «Ещё одна подписка» → 3 дня без карты
4. «На YouTube бесплатно» → протокол + адаптация, не плеер
5. «Нет времени разбираться» → 3 вопроса → трек сразу

## Ключевые продуктовые решения

| Решение | Детали |
|---------|--------|
| Платежи | ЮKassa + СБП (не Stripe/Paddle) |
| Воронка | Telegram Login Widget → 3-дневный триал («три ночи») → оплата СБП через бота или лендинг |
| Авторизация MVP | Telegram Login Widget. Email — в V2, когда появится VK-трафик без VPN |
| Данные пользователя | Только telegram_id + subscription_status + usage_data. Телефон/email не собираем. |
| Сессия | httpOnly cookie (не localStorage — уязвимость к XSS) |
| Telegram-бот | Канал удержания: напоминания о сессии, реактивация, конверсия в день 3 триала |
| Доставка | Веб + VPS (FileResponse). НЕ через Telegram (медиатрафик душится) |
| Звук (MVP) | Creative Commons ambient + DSP-синтез для частот. Suno — только для черновиков до решения суда (июль 2026) |
| Трафик | VK Клипы (#1 приоритет), далее: Instagram / TikTok / YouTube Shorts (через VPN) |
| Правовое | Без «научно доказано», без сольфеджио/резонанса Шумана как продающих аргументов |

## Звуковая начинка (зафиксировано)

Используемые частоты:
- Дельта 1–3 Гц → глубокий сон
- Тета 4–7 Гц → засыпание, восстановление
- Альфа 8–12 Гц → снятие стресса
- Низкая Бета 10–14 Гц → спокойный фокус
- Бета 15–18 Гц → глубокая работа

НЕ используем: высокая Бета 20–30 Гц, Гамма/40 Гц, сольфеджио.  
Бинауральные биты — только в наушниках. Изохронные тоны — работают без наушников.

### Блок 5 — Техническая архитектура ✓ ЗАФИКСИРОВАНО v2.0
Файл: `docs/architecture.md`

Стек выровнен по Nura — всё в коде, никаких платных внешних сервисов:
- Фронтенд: vanilla HTML/CSS/JS (как Nura), без фреймворков и бандлера
- Backend: FastAPI 0.115 + SQLAlchemy 2.0 async + Pydantic 2
- БД: PostgreSQL 16 + Alembic миграции
- Redis: FSM для aiogram бота
- Деплой: Docker Compose (4 контейнера: api, bot, postgres, redis) + Nginx
- Auth: Telegram Login Widget + HMAC верификация + httpOnly cookie
- Аудио: FileResponse через FastAPI, файлы на VPS (не CDN на MVP)
- Платежи: yookassa 3.1 + HMAC webhook (паттерн из Nura)
- Порядок разработки: 9 шагов от структуры до продакшн-деплоя

### Блок 6 — Биллинг ✓ ЧЕРНОВИК ГОТОВ
Файл: `docs/billing.md`

- Recurring на MVP: ручное продление + проактивные напоминания через бота (без автосписания)
- Автосписание (payment_method_id) — V2, после 100+ подписчиков
- Планировщик: apscheduler без Celery (trial конверсия день 3 в 18:00, напоминания за 3 дня, downgrade в 02:00)
- Возвраты: ручные в первые 24 часа если не открывал сессий, через ЮKassa Refund API

### Блок 7 — Продакшн звука ✓ ЧЕРНОВИК ГОТОВ
Файл: `docs/audio-production.md`

- 15 треков на старте (4 категории, разные длительности: 15/20/30/45/60 мин)
- DSP-слой: Python (numpy + scipy + soundfile) — скрипт в tools/
- Ambient-слой: Creative Commons на MVP (CC0 / CC BY только; Suno — черновики до решения суда июль 2026)
- Микширование: Audacity вручную (ambient -18 LUFS, DSP -24 LUFS, итог -14 LUFS)
- 8 бинауральных треков (наушники обязательны) + 7 изохронных (без наушников)
- Метаданные: audio/catalog.json — источник правды для плеера
- Аудиофайлы в .gitignore, деплой через rsync отдельно от кода

### Блок 8 — Метрики MVP ✓ ЗАФИКСИРОВАНО
Файл: `docs/metrics-mvp.md`

- Воронка целиком: просмотр → лендинг → триал → оплата → удержание
- Критерии «гипотеза подтверждена»: Trial→Paid >20%, D30 >35%, CAC <200 ₽ на когорте 50+
- Критерии «стоп — пивот»: Trial→Paid <10% после 100 триалов
- Инфраструктура окупается с 2–3 подписчиков (VPS ~925 ₽/мес ÷ 476 ₽ чистых)
- SQL-запросы для ключевых метрик прямо из БД (без внешней аналитики)
- Ориентиры рынка: Brain.fm, Calm, типичный B2C wellness SaaS
